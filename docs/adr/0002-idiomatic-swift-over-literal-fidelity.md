# Match AshTypescript's features idiomatically, not literally

AshSwift's goal is to give a Swift app a plug-n-play, type-safe way to talk to an Elixir/Ash backend — the same goal AshTypescript serves for TypeScript. We aim to match as many AshTypescript features as possible, but where TypeScript and Swift differ we choose the **idiomatic Swift** expression of the feature over a literal reproduction of the TypeScript API. Example: TypeScript narrows return types to the requested fields via conditional/mapped types over array literals; Swift's nominal type system can't do that for runtime field lists, so we express field selection idiomatically (e.g. Codable structs + typed queries) rather than building a baroque generics/parameter-pack emulation. This principle governs tie-breaks throughout the project.
